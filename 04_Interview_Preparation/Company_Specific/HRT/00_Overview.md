# HRT (Hudson River Trading) Interview Preparation

## Company Overview

**Type:** Proprietary trading / Market making  
**Founded:** 2002  
**HQ:** New York  
**Focus:** High-frequency trading, technology-driven

## Culture

- Engineering excellence
- Low-latency obsession
- Small teams, high impact
- Flat hierarchy
- Technical meritocracy

## Interview Process

### Typical Flow
1. HackerRank / Online coding (1-2 hours)
2. Phone screen (45-60 min)
3. Onsite / Virtual (4-6 hours)
   - Multiple coding rounds
   - System design
   - Market microstructure
   - Brain teasers

## Key Focus Areas

| Area | Weight | Notes |
|------|--------|-------|
| Coding | 35% | Algorithms, data structures |
| System Design | 25% | Low-latency, HFT systems |
| Market Microstructure | 20% | LOB, execution |
| Math/Probability | 15% | Quick mental calculations |
| Behavioral | 5% | Team fit |

## Essential Topics

### Low-Latency Systems
- Network optimization
- Memory management
- Cache efficiency
- Lock-free programming
- FPGA basics

### Market Microstructure
- Limit order books
- Market making mechanics
- Order flow
- Price discovery
- Execution algorithms

### Data Structures & Algorithms
- Tree structures (BST, heap)
- Hash tables
- Graph algorithms
- String algorithms
- Optimization

### Coding (Often C++, but Python accepted)
- Clean, efficient code
- Optimization techniques
- Memory efficiency
- Concurrency

## Sample Questions

1. Implement an order book data structure
2. Design a low-latency trading system
3. What causes latency in a trading system?
4. How does market making work?
5. Optimize this code for speed

## Tips for Success

1. **Speed matters**: Both solving and code execution
2. **Think about latency**: Always consider performance
3. **Know microstructure**: Understand how markets work
4. **Clean code**: Even under time pressure
5. **Ask about constraints**: Memory, latency, scale
