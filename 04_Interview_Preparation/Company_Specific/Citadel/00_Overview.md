# Citadel Interview Preparation

## Company Overview

**Type:** Hedge fund + Market maker (Citadel Securities)  
**Founded:** 1990 (Ken Griffin)  
**HQ:** Chicago, Miami  
**AUM:** ~$60+ billion (hedge fund)

## Two Businesses

### Citadel (Hedge Fund)
- Multi-strategy hedge fund
- Global macro, equities, fixed income, commodities, quant
- Research-driven

### Citadel Securities (Market Maker)
- Largest market maker in US equities
- Options, ETFs, fixed income
- Technology-driven

## Interview Process

### Typical Flow
1. Online assessment / HackerRank (1-2 hours)
2. Phone screen (45-60 min)
3. Virtual/Onsite superday (5-7 hours)
   - Multiple technical rounds
   - System design
   - Coding deep dives
   - Behavioral

## Key Focus Areas

| Area | Weight | Notes |
|------|--------|-------|
| ML/Statistics | 30% | Depth on algorithms |
| Coding | 30% | Data structures, algorithms, clean code |
| System Design | 20% | Large-scale data, latency |
| Finance | 15% | Markets, strategies |
| Behavioral | 5% | Culture fit |

## Essential Topics

### Machine Learning (Depth Required)
- All supervised algorithms in detail
- Feature engineering at scale
- Model validation (purged CV, multiple testing)
- Overfitting prevention
- Production ML systems

### System Design
- Feature store design
- Real-time prediction systems
- Data pipelines at scale
- Latency optimization
- Distributed computing

### Coding
- Algorithm optimization
- Data structures (trees, heaps, graphs)
- Pandas/NumPy efficiency
- Code quality and testing

### Statistics
- Hypothesis testing
- Regression analysis
- Time series
- Causal inference

## Sample Questions

1. Design a feature store for a quant research team
2. How do you handle concept drift in production?
3. Implement purged k-fold cross-validation
4. Optimize this pandas code to handle 1TB of data
5. Explain the bias-variance tradeoff

## Tips for Success

1. **Technical depth**: Know algorithms inside-out
2. **Scale thinking**: Always consider large data
3. **Production mindset**: Think about deployment
4. **Code quality**: Write clean, tested code
5. **Ask questions**: Clarify requirements before solving
